## 1. 系統概述

### 1.1 專案背景

非營利組織管理系統，轉換為現代化 **Web-Based + PostgreSQL + Python** 全新架構。同步規劃手機網頁 APP 應用，整合 Google 帳號認證與 LINE 應用，並透過 QR Code 建立分享帳號機制。

### 1.2 系統範圍

| 項目 | 內容 |
|:-----|:------|
| **系統名稱** | NPOSystem — 非營利組織管理系統 |
| **目標使用者** | 內部行政人員、管理階層 |
| **主要功能** | 捐款管理、組織黨員管理、報表查詢 |
| **架構** | Web-Based (瀏覽器)，後續擴展至手機網頁 APP |
| **資料遷移** | 遷移至 PostgreSQL |

## 6. 系統整合需求

### 6.1 Google 帳號認證整合

| 項目 | 說明 |
|:-----|:------|
| **功能** | 使用者可使用 Google 帳號登入系統 |
| **技術** | Google OAuth 2.0 (OpenID Connect) |
| **範圍** | 取得使用者 email、姓名、頭像 |
| **綁定** | 首次 Google 登入需綁定既有員工帳號 |
| **優先級** | P1 |

**流程：**
```
使用者點擊「Google 登入」
        ↓
Google 授權頁面 (選擇帳號)
        ↓
系統接收授權碼 → 交換 Access Token
        ↓
驗證 ID Token → 取得使用者資訊
        ↓
比對系統員工 → 建立 Session → 登入成功
```

### 6.2 LINE 應用整合

| 項目 | 說明 |
|:-----|:------|
| **功能** | 使用者可透過 LINE 帳號登入；系統可發送 LINE 通知 |
| **技術** | LINE Login + LINE Messaging API |
| **範圍** | 登入認證、捐款/扣款通知、活動通知 |
| **優先級** | P2 |

**可能應用場景：**
- 📢 捐款確認通知
- 📢 扣款成功/失敗通知
- 📢 活動邀請通知
- 📢 黨費繳交提醒

### 6.3 QR Code 分享帳號機制

| 項目 | 說明 |
|:-----|:------|
| **功能** | 產生 QR Code 用於分享特定資料（捐款人、黨員） |
| **技術** | Python qrcode 套件 + Pillow |
| **應用場景** | 分享捐款人/黨員資料連結、活動報名 QR Code |
| **優先級** | P2 |

## 7. 非功能需求

### 7.1 技術指標

| 項目 | 要求 | 說明 |
|:-----|:-----|:------|
| **瀏覽器支援** | Chrome / Firefox / Edge (最新版) | 不支援 IE |
| **行動裝置** | RWD 響應式設計 | 手機網頁 APP 架構 |
| **API 回應時間** | < 500ms (一般查詢) | 大量報表可接受 < 3s |
| **同時上線人數** | 初期 20~50 人 | 架構支援水平擴展 |
| **資料備份** | 每日自動備份 DB | PostgreSQL pg_dump |

### 7.2 安全需求

| 項目 | 說明 |
|:-----|:------|
| **認證方式** | JWT Token + Redis Session |
| **密碼儲存** | bcrypt 雜湊 |
| **API 安全** | CORS 限制、Rate Limiting |
| **資料權限** | 角色基礎存取控制 (RBAC) |
| **HTTPS** | 正式環境強制 HTTPS |

### 7.3 資料遷移需求

| 項目 | 說明 |
|:-----|:------|
| **來源** | Excel 資料表 |
| **目標** | PostgreSQL 16 |
| **遷移工具** | Python 腳本 (mssql_to_pg.py) |
| **資料驗證** | 遷移後需逐表比對筆數正確性 |
| **優先級** | P0 (上線前必須完成) |
